PROJECT VELA

Model Responsibilities

  • Challenge assumptions
  • Choose and explain designs
  • Adapt tone without losing honesty

Code Responsibilities

  • Parse, validate, schedule, replay
  • Compile, lint, test, diff
  • Enforce schemas and permissions
  • Capture provenance automatically

Native Dataset Records

  • Task → patch
  • Diagnostic → fix
  • Review → revision
  • Decision → rationale

Merge Boundary

  • Routine: green checks may auto-merge
  • Architecture/security/identity: user approval
  • No secrets or private prompts in corpus
  • Main remains releasable
